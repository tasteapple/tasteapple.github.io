Project Detail / 프로젝트 상세

OpenClaw Team Project

OpenClaw를 직접 구성하고 사용하면서, AI agent 환경의 취약점과 동작 특성을 실습 중심으로 검토해본 팀 프로젝트.

프로젝트 목록으로

Overview / 개요

이 프로젝트는 OpenClaw를 직접 써보고 싶다는 호기심에서 시작됐다. 최근 온디바이스 AI와 agent 환경이 점점 늘어나고 있고, 앞으로는 일상적인 형태로 자리 잡을 가능성이 높다고 생각했기 때문에, 단순히 소개만 보는 것이 아니라 직접 구성하고 테스트해보는 쪽으로 접근했다.

그 과정에서 단순 사용 경험에만 머무르지 않고, 어떤 취약점이 있는지, 어떤 공격이 통하는지, 모델 버전이나 OpenClaw 버전에 따라 어떤 차이가 생기는지를 확인해보는 쪽으로 프로젝트가 확장됐다.

Core Focus / 핵심 내용

  • OpenClaw 직접 구성 및 사용
  • 취약점 테스트
  • RCE와 프롬프트 인젝션 시도
  • 모델 버전별 동작 차이 확인

Why I Built It / 만든 이유

가장 큰 이유는 OpenClaw를 직접 써보고 싶었기 때문이다. 요즘 AI agent가 점점 더 주목받고 있고, 결국에는 온디바이스 환경에도 AI들이 탑재되고 agent 활용이 일상화될 것이라고 생각했다. 그래서 실제 환경을 직접 구성하고, 동작 방식과 보안 측면을 스스로 확인해보고 싶었다.

My Role / 역할

이 프로젝트에서 나는 기획, OpenClaw 구성, 그리고 취약점 테스트를 맡았다. 단순 사용이 아니라 직접 환경을 만들고, 테스트 시나리오를 잡고, 어떤 부분이 동작하고 어떤 부분이 막히는지 검토하는 쪽에 집중했다.

How I Want It Framed / 어떻게 소개하고 싶은가

이 프로젝트는 ‘OpenClaw 기반 팀 프로젝트’이자 ‘AI agent 활용 프로젝트’로 소개하는 것이 가장 잘 맞는다. 핵심은 단순히 OpenClaw를 사용해본 것이 아니라, 실제로 구성하고, 테스트하고, 취약점 관점에서 검토한 경험까지 담겨 있다는 점이다.

What Stands Out / 강조할 포인트

  • OpenClaw를 직접 구성하고 실제로 사용해본 프로젝트
  • RCE, 프롬프트 인젝션 등 취약점 관점 테스트를 수행
  • OpenClaw 버전과 모델 버전에 따라 동작 차이를 관찰
  • 단순 실습이 아니라 실제 구현과 산출물 정리까지 포함
